# My 90-Day DevOps Learning Plan

## Introduction

*   **My Understanding of DevOps and Cloud Engineering:** My understanding is that DevOps is a cultural and professional movement that emphasizes collaboration and communication between software developers (Dev) and IT operations (Ops) to automate the software delivery lifecycle. The goal is to deliver high-quality software faster and more reliably. Cloud Engineering provides the on-demand, scalable, and manageable infrastructure (like AWS, Azure, GCP) that makes modern DevOps practices possible.

*   **My "Why" for Learning:** I am starting this journey to transition into a high-growth technology role that aligns with my passion for building efficient, automated systems. I am driven by the challenge of bridging the gap between development and operations to create resilient and scalable applications that solve real-world problems.

*   **Current Level:** I am a fresher with a foundational knowledge of IT concepts, eager to build the practical, in-demand skills required to become a successful DevOps Engineer.

## The Destination: My 90-Day Goals

1.  **Containerize and Deploy a Web Application:** Gain proficiency in Docker by containerizing a multi-service web application and managing it effectively using Docker Compose.
2.  **Build a Full CI/CD Pipeline:** Implement an end-to-end CI/CD pipeline using Jenkins that automatically builds, tests, and deploys a containerized application to a Kubernetes cluster upon a code change in a Git repository.
3.  **Automate Cloud Infrastructure:** Deploy a production-grade, scalable web application on AWS by provisioning the entire infrastructure (VPC, EC2 instances, Load Balancers) using Terraform and implementing basic monitoring with Prometheus and Grafana.

## The Engine: Core Skills to Build

1.  **Infrastructure as Code (IaC):** Develop strong skills in using Terraform to automate the provisioning and management of cloud infrastructure, ensuring consistency and repeatability.
2.  **CI/CD Pipeline Automation:** Master the art of creating, managing, and troubleshooting robust, automated pipelines using Jenkins, integrating tools like Git, Docker, and Kubernetes.
3.  **Container Orchestration with Kubernetes:** Gain the ability to deploy, manage, and scale containerized applications on Kubernetes, with a deep understanding of core concepts like Pods, Deployments, Services, and ConfigMaps.

## The Fuel: Consistency & Time Budget

*   **My Commitment to Consistency:**
    *   **Time-Blocking:** I will dedicate a non-negotiable time slot every day to focus solely on learning and hands-on practice.
    *   **Learn in Public:** I will share my daily progress and key learnings on LinkedIn using the #90DaysOfDevOps hashtag to maintain accountability and build a professional network.
    *   **Hands-On First:** I will prioritize practical labs and projects for at least 70% of my study time, ensuring I am building real skills, not just consuming content.
    *   **Weekly Reviews:** Every Sunday, I will review my progress, identify areas of weakness, and plan the specific tasks for the upcoming week.

*   **Weekly Time Budget:**
    *   **Weekdays:** 2.5 - 3 hours per day
    *   **Weekends:** 4 - 6 hours per day
    *   **Total Weekly Commitment:** Approximately 25-30 hours

## The Roadmap: A 90-Day Breakdown

This roadmap is structured based on the "DevOps Zero to Hero" syllabus to ensure a logical progression from fundamentals to advanced, real-world projects.

### **Month 1: The Foundation (Days 1-30)**

*   **Weeks 1-2: Linux & Shell Scripting:**
    *   **Focus:** Master the Linux command line, file system, user/permission management, and basic networking.
    *   **Project:** Write shell scripts to automate system backups and log analysis.
*   **Weeks 3-4: Version Control with Git & GitHub:**
    *   **Focus:** Deep dive into Git fundamentals, including branching, merging, and rebasing. Learn advanced concepts like resolving merge conflicts and using SSH keys.
    *   **Project:** Manage a project repository on GitHub, using feature branches and pull requests for collaboration.

### **Month 2: Core DevOps Tooling (Days 31-60)**

*   **Weeks 5-6: Docker & Containerization:**
    *   **Focus:** Understand Docker architecture, create custom images with Dockerfiles, and manage multi-container applications with Docker Compose.
    *   **Project:** Containerize a two-tier application (e.g., a web front-end and a database).
*   **Weeks 7-8: CI/CD with Jenkins & Introduction to Kubernetes:**
    *   **Focus:** Set up Jenkins, create a "Pipeline as Code" using a Jenkinsfile, and begin exploring Kubernetes architecture, Pods, and Deployments.
    *   **Project:** Build a CI pipeline that automates the building of a Docker image and pushes it to a registry.

### **Month 3: Cloud, IaC, and Projects (Days 61-90)**

*   **Weeks 9-10: Infrastructure as Code (Terraform & Ansible):**
    *   **Focus:** Learn to define and provision cloud infrastructure using Terraform. Understand Ansible for configuration management.
    *   **Project:** Write Terraform code to create a basic network and server setup on AWS.
*   **Weeks 11-12: Advanced Cloud, Monitoring & Final Project:**
    *   **Focus:** Explore core AWS services (EC2, S3, VPC, IAM), implement monitoring with Prometheus & Grafana, and integrate all learned skills.
    *   **Final Project:** Deploy the containerized, two-tier application from Week 6 onto a Kubernetes cluster running on AWS infrastructure provisioned by Terraform, with the entire process automated by a Jenkins CI/CD pipeline.
